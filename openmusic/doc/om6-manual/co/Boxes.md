<div class="headCo">

# <span>Boxes</span>

<div class="headCo_co">

<div>

<div class="infobloc">

<div class="txt">

Boxes are the main elements at stake in the composition of a patch. Each
box type refers to a given functional component : a function, an object,
a value, a program or sub program, and so on. Among these,
<span id="i0" class="defRef_ul"><span>function</span></span><sup>[<span>\[</span>1<span>\]</span>](#kFootBsktc1872)</sup>
boxes, and object or <span id="i1" class="defRef_ul"><span>factory
boxes</span></span><sup>[<span>\[</span>2<span>\]</span>](#kFootBsktc2025)</sup>
appear as major tools. They also are the most numerous ones. This
section lists and describes each of these OM box types, their behaviour
and use.

</div>

</div>

</div>

</div>

</div>
